[NOTE]: Text Detection with Online Tracking

Posted on 2022-02-28 Edited on 2022-03-05 In Reading Notes Views: Views: Valine:

这是一篇较早的文字追踪文章，是百度和自所联合发表的

Abstract

SDE模型。对于视频文字检测，使用tracking的方式可以解决视频文字的一些难题（e.g. motion blur），该文章提出一种end2end的方式，在detection分支，使用ConvLSTM结构来获得空间上的信息。在追踪端，将追踪问题看作文字关联的问题。而这样将文字追踪看作detection和tracking (association)两部分也是大部分文章所使用的方式.

Introduction

传统的文字追踪网络通常采用two-step的方法：先在每一帧检测一下，再传入到关联网络中，之前看的那篇Video TextSCM就是采用这样一个方式

  • 在时间维度上，每一帧图片其实并没有做到充分的训练
  • 两阶段训练预测，导致额外的计算cost
  • 容易suboptimal

所以文章使用ConvLSTM来做检测，企图对图片在时间维度有充分的利用。然后使用appearance-geometry来进行匹配

Method

Overview

使用ResNet50 + U-Net来做backbone，然后使用EAST的方法来做（生成score map 和 geo map）。然后加上ConvLSTM。detected proposals和当前帧的common features作为descriptor generation输入。descriptor generation的输出进入GRU，然后用上一帧的输出和这一帧GRU的输入做similarity matrix，得到轨迹

Text Detection Branch

backbone + ConvLSTM cell + EAST生成map

Text Tracking Branch

使用NMS来获得top-K个proposals，即k个bbox。然后和feature map（ConvLSTM）的输出共同作为Descriptor Generation的输入，得到相似矩阵后，使用KM算法来获得匹配对
