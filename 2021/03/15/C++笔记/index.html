C++ 笔记 (C++ Note)

做项目时Python用到的过多，C++逐渐生疏。当再用C++时会出现一些错误，本笔记在复习C++之余，也尝试的寻找和Python的较大差异之处，方便以后加强记忆。有疑问可在评论区中提问

During project, I preferred to use Python, forgetting some methods and usage in C++. Thus, when I utilize C++ again, there are some problems and mistakes. This note acts as a record of my revision and also illustrates some difference between C++ and Python to make switching languages (Python and C++) easy in the future. Hoping for your comments! : ). The whole note is written in Chinese

内存四大分区

代码区

定义：存放二进制代码，由操作系统进行管理

共享，只读：

共享：运行多少次同一程序不会再内存中生成多份二进制

只读：可结合Linux中的C++编译过程（gcc为例子）：

  • 预处理,生成 .i 的文件
  • 将预处理后的文件转换成汇编语言, 生成文件 .s
  • 有汇编变为目标代码(机器代码)生成 .o
  • 连接目标代码, 生成可执行程序

只读就是运行可执行程序进入内存后无法更改。

全局区

  • 全局变量：写在函数体外的变量 （相关）
  • 静态变量：static关键词修饰的关键字
    • static修饰全局变量：作用域仅在该源文件中
    • static修饰局部变量：只会被定义一次，不会重复初始化（符合全局区在代码执行前的特征）
  • 常量：
    • 字符串常量：和Python相比，字符串在C++中是一个常量，而Python中的字符串更像是对应的STL 中的 String
    • 其他常量：const 修饰的全局常量
      • const 修饰的全局变量：与普通全局变量相比，全局常量无法修改
      • const修饰的局部变量：不在全局区，所以可能会被修改（不经常用）

  • 引用必须被初始化（有点像防止指针变成野指针）
  • 一旦初始化后，就不可以更改了

引用做函数的返回值

  • 不要返回局部变量的引用 （栈区）：一般返回的是static修饰的局部变量的引用
  • 函数的调用可以作为左值：即函数可以写到等号的左边

重载 Overide

  • 函数重载（类方法重载）
    • 同一作用域下
    • 函数名称相同
    • 函数参数类型不同，个数不同或顺序不同

      当有默认参数的时候，要注意会不会发生歧义

类

封装，继承，多态

类的语法总览

在Python中，所有的属性写在__init__()函数中，默认为public权限。若想改成private权限，可通过更改变量名字

struct和class的区别

**唯一的区别在于默认的访问权限不同

  • struct 默认为公共
  • class 默认权限是私有

访问权限

访问权限是指的属性和行为（方法）的权限

  1. public 公共权限
    成员类内可以访问，类外也可以访问
  2. protexted 保护权限
    类内可以访问，类外不可以访问，继承时，子类可以访问父类
  3. private 私有权限
    类内可以访问，类外不可以访问，继承时，子类不可以访问父类
    例子 IDE一般会提示你访问到了私有权限/保护权限
    所有权限该类内随意访问
    总结：
  • 成员属性一般设置为私有属性，更改，初始化，读通过public的方法

构造函数和析构函数

构造函数：主要作用在于创建对象时为对象的成员属性赋值，实例化类的时候执行
析构函数：系统自动调用，清理（主要是清理类内在堆区声明过的变量）
构造函数和析构函数都赋予public
语法


构造函数可以重载，析构函数不可以

构造函数的分类及调用

两种分类方式：

  • 按参数分为： 有参构造和无参构造
  • 按类型分为： 普通构造和拷贝构造

三种调用方式：

  • 括号法
  • 显示法

  • 隐式转换法

    同理，当按照值返回类的对象的时候，也会调用拷贝构造（return一个类的时候）

  • 调用

  1. 括号法 括号法是较常用的方法， 类似Python
  2. 显示法
  3. 隐式转换法 默认情况下，C++编译器至少给一个类添加三个函数
    默认构造函数，默认析构函数，默认拷贝构造函数（浅拷贝）
    当提供拷贝构造函数后，一定还要提供普通构造函数
    拷贝构造函数可以拷贝私有属性

    深拷贝与浅拷贝

    浅拷贝：简单的赋值拷贝操作
    深拷贝：在堆区重新申请空间，进行拷贝操作
    浅拷贝就是对于变量的拷贝。对于指针类型的属性（这个指针类型的属性在构造函数中使用new创建在堆区）。当有指针类型的属性时，记得在析构函数里面delete并将指针赋值为Null。但当使用默认拷贝构造函数时，在析构函数释放堆区内存时会导致重复释放（同一地址）。
    深拷贝就是为了解决这一问题，在堆区重新申请空间，进行拷贝操作。相当于在自己实现拷贝构造函数时，使用 再写拷贝构造函数时，无论是公共属性还是私有属性，都可以直接使用 . 来进行读值。所以，前文提到的“类内”，指的是同属一个类，就是类的名字只要是一个就行（不是实例化的名字）。只要同属一个类就可以不用区分同一个类的不同对象

静态成员

静态成员变量：

  1. 在编译阶段分配内存
  2. 类内声明，类外初始化
  3. 所有对象共享同一份数据

静态成员函数：

  1. 程序共享一个函数
  2. 静态成员函数只能访问静态成员变量

因为是共享一个函数，不属于任何一个对象，所以可以通过普通方式来调用，也可以直接像调用普通函数一样调用


静态成员函数同样有public和private两种权限

C++对象所占内存

  • 空类占1字节
  • 非空类占非静态成员变量的大小
  • 成员函数不占类对象内存

成员函数不占用内存是因为使用了this指针，同Python中的self

this 指针

使用方法：this->...

  1. 解决名称冲突
  2. 返回对象本身 return *this
    在Python的内置方法中，所有的方法都会返回类本身，即return self。在C++中，使用return *this 返回值为类的引用（链式编程思想）
    属性在类内调用时，都写this->
    this指针是一个指针常量
    ClassName* const this

  • 常对象：
