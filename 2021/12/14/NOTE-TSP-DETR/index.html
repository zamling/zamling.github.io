[NOTE]: TSP-DETR

Posted on 2021-12-14 In Reading Notes Views: Views: Valine:

论文的题目是Rethinking Transformer-based Set Prediction for Object Detection

Abstract

指出DETR将一个检测问题看成了set prediction问题（有点一步到位的感觉）

Set prediction is about learning to predict a collection of unordered variables with unknown interrelations

提出了TSP-FCOS和TSP-RCNN两个解决收敛速度慢的方法

Introduction

set prediction的方法主要就是解决了后处理，如NMS的方法的省略。根据使用NMS的方法，如Faster-RCNN，究其原因，还是框和框之间在网络中没有任何联系的，也就是每个框都没有综合获得过一个全局的信息。这也使得使用self-Attention。在实验中，作者发现，主要是cross-attention和匈牙利匹配使得收敛变慢，所以提出了以下想法

  • 删除decoder部分
  • 新的Feature of Interest (FoI)来做multi-level feature
  • 新的二分图匹配

什么导致了收敛缓慢

匈牙利为什么导致收敛缓慢

  • 初始化是random的
  • 匹配不是很稳定，容易被noise干扰

Matching distillation来进行验证

用了一个与训练好的DETR模型的matching部分来做assignment，发现确实收敛加快了（肯定会加快，但感觉加快的不多啊，感觉不是匈牙利的锅），只对训练前几个epoch有用。感觉这个方法有点unfair

Attention modules的问题

由于初始的Attention大多是uniform分布的（这也是序列不能过长的原因），而最终的attention是一个非常sparse的，所以这个一个sparse的过程是否会导致收敛缓慢呢？

作者着重提出了cross attetnion，因为cross attention将encoder得到的information给了decoder，如果这些信息不准确，就寄了（这也是DETR小目标检测不准确的原因，经过Backbone，信息丢失，到encoder输出上就没有了小目标的信息了）。

这里使用了一个negative entropy来看稀疏情况。发现尤其第一层的cross attention，几十个epoch之后稀疏性都没有收敛。所以网络也没有收敛

Decoder的重要性

去掉decoder后，收敛加快。在小目标的检测上也有提高，但在大目标上就寄了。作者也提出了自己的一个解释：大目标有太多的可匹配的feature points了，所以在encoder中的attention稀疏之后很难关注到这么多点了
