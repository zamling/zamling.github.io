随后对位置编号进行Normalization，将其归一化到$0-2\pi$ 。具体步骤为除以最大有效编码，乘以$2\pi$.

这里在拼接tensors时使用了stack，stack和cat主要区别为：

  • stack先在参数dim扩展出来一个维度，然后在这个新的维度进行拼接
    • 两个（2，2）维度matrix在dim=0拼接将变成（2，2，2）
  • cat的tensors必须维度个数一样，然后在原有的dim维度进行拼接
    • 两个（2，2）维度matrix在dim=0拼接将变成（4，2）

flatten的参数是从第几维开始，后面开始flatten

他这样子写可以让原顺序不变，而直接cat第四个维度会导致奇偶顺序改变

PositionEmbeddingLearned

两个维度的值直接使用可学习的embedding来进行，将位置直接映射成一个num_pos_feats维度， 然后对每个坐标$(h_i,w_i)$，一半的embedding来自h的embedding number，一半来自w的embedding number

nn.MultiheadAttention中的mask有两种：

  • key_padding_mask
  • attn_mask
    官方源码中，两个mask分别为
  • key_padding_mask: if provided, specified padding elements in the key willbe ignored by the attention. When given a binary mask and a value is True,the corresponding value on the attention layer will be ignored. When givena byte mask and a value is non-zero, the corresponding value on the attentionlayer will be ignored。即在计算QV内积时，会将mask的给去掉 $(N, S)$ [batch size, key length]
  • attn_mask: 2D or 3D mask that prevents attention to certain positions. A 2D mask will be broadcasted for all the batches while a 3D mask allows to specify a different mask for the entries of each batch. $L, S$ [query length,key length]

实现部分为

看完这个源码，其实可以发现两个mask其实都是在mask QV的内积，而且每个内积都会mask两遍，所以其地位是等同的。由于key_padding_mask的形状受限的原因，一般作为一个padding mask，而attn_mask是接受了Q V分别的mask，可以接受Q,V不等大的padding mask（如encoder decoder连接时的mask），或者也可以用来做sequence mask（直接传入一个下三角矩阵）

在DETR中，使用情况见下节

TransformerEncoderLayer & TransformerEncoder

单层encoder的结构
输入：src，及 src_key_padding_mask (mask)

src_mask是None

推理过程为

  • q,k是对src加positional encoding的结果，，v是src
  • 经过self-attention层（其中，rc_key_padding_mask作为key_padding_mask）
  • Norm(src+dropout(src))
  • 两层FC层，夹着一个dropout层
  • Norm(src+dropout(src))
  • end

另一个版本是将LayerNorm层提到每一个子层的前面（Multi-Attention-layer & FFN）

NOTE：如果采用的是在所有前面采用LayerNorm，要在最后再加上一个norm

TransformerDecoderLayer & TransformerDecoder

单层decoder的结构
输入：tgt，memory, tgt_mask, memory_mask, tgt_key_padding_mask, memory_key_padding_mask ,pos, query_pos
memory_key_padding_mask是mask，其余的mask均为0

其中tgt为输出，pos是encoder的位置编码，memory为encoder的输出，query_pos是object queries（经过可学习的位置编码）

推理过程为

  • q,k是对tgt加query_pos的结果，v是tgt
  • 经过self-attention层（其中，tgt_mask作为attn_mask，tgt_key_padding_mask作为key_padding_mask）
  • Norm(tgt+dropout(tgt))
  • 经过一个self-attention层，其中q是tgt+object queries，k是memory+pos，v是memory（其中，memory_mask作为attn_mask，memory_key_padding_mask作为key_padding_mask）
  • Norm(tgt+dropout(tgt))
  • 两层FC层，夹着一个dropout层
  • Norm(src+dropout(src))
  • end

另一个版本是将LayerNorm层提到每一个子层的前面（Multi-Attention-layer & FFN）

整个Transfomer的推理过程

输入src （经过backbone后的值），mask，query_embed，pos_embed
其中query_embed [N, hidden_dim]
输出:

  • hs: decoder输出， object queries同形状的，最后在dim=0的位置扩一个维度（原因是可以添加一个每一层decoder输出的维度，当return_intermediate设置为false时，就是只输出最后一个layer的输出） [num_layer,batch size,N, embed_size]
  • encoder的输出，格式转变成backbone的输出格式

推理过程

  • 将src的形状从NxCxHxW 变成 HWxNxC
  • 将pos_embed形状从[N,embed,H,W]变成[HW,N,embed]
  • encoder中的src_key_padding_mask和decoder中的memory_key_padding_mask均是mask（因为mask都是从图片的那个NestedTensor来的）尤其是对于encoder和decoder连接时的memory_key_padding_mask来说，相当于tgt_key_padding_mask等于0，也就是tgt是没有padding的，所以这里只mask memory部分就行了）
  • 注意tgt赋值为了query_embed size的0，也就是说输出形状和object queries一样
